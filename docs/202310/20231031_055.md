# OpenAI 将发布 Stateful API，降低 GPT API 调用成本

> 原文：[`www.yuque.com/for_lazy/xkrm14/ktwme7w7un4mzgxr`](https://www.yuque.com/for_lazy/xkrm14/ktwme7w7un4mzgxr)

正文：

路透社报道说，OpenAI 将在 11 月 6 日的开发者大会上发布 Stateful API ，此举讲大幅度降低开发者调用 GPT API 成本。 要理解
Stateful API ，先得知道目前的 API 是 Stateless 的。 什么意思呢？ 就是说，目前的 API
是无状态的，大模型每次运算都当做一次全新的运算，所以状态需要开发者维护。 更直白的说，就是目前的 API
记不住对话上下文，需要开发者自己把对话的历史聊天记录传进去。 每次用户发送一句新的对话，开发者都需要把此前三轮的问和答传进去。
这就导致传输的内容大量的重复，也导致消耗了大量的 token ，也就浪费了大量金钱。 不过，如果只是把 Stateless 改为 Stateful
，我感觉并不能把费用降低到原来的二十分之一。 因为有大量任务本身并不需要传入历史对话，如翻译、打标签、总结等等，每次只需要处理当前传入的即可。
所以要想降低费用，价格也得跟着降低。 最有可能降的就是 GPT4 API 的价格了，目前 GPT3.5 的价格已经很低了，降价了也感受不明智，但是 GPT4
的价格是 GPT3.5 的 40 倍左右，如果能够降到 GPT3.5 的价格，那么所有基于 GPT API 做的应用都可以不用考虑省钱了，也就是很多事情的
ROI 都能算得过来了，那么商业模式都将改变。 所以，开发者们，可以提前想一想，如果 GPT4 API 价格真的降低了，你可以做哪些事情？
